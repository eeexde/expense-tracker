# Kuripot v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Offline-first Android expense/income tracker (APK) for Filipino users: buckets, recurring + installment auto-posting, utang tracking, receipt OCR, monthly stats.

**Architecture:** Expo dev-build app. Pure-TS core logic modules (money, recurring engine, receipt parser) tested in Jest with better-sqlite3 standing in for expo-sqlite. UI layer in Expo Router tabs reads/writes through a repository layer over Drizzle ORM. Catch-up job runs on app open.

**Tech Stack:** Expo SDK 53+, TypeScript, expo-sqlite, Drizzle ORM, Expo Router, expo-camera, @react-native-ml-kit/text-recognition, expo-notifications, react-native-gifted-charts, Jest + better-sqlite3 (tests).

## Global Constraints

- All money amounts are **integer centavos** everywhere. Display via `formatPeso`.
- Balances are always derived by query; never stored.
- Spec: `docs/superpowers/specs/2026-07-03-expense-tracker-design.md`.
- Node 20+. App id `com.edrian.kuripot`.
- Auto-posted transactions carry `recurringId` or `installmentId`.
- Commit after every green test cycle.

### Task 6: Receipt parser

**Files:** Create `src/lib/receiptParser.ts`, `src/lib/receiptParser.test.ts`.

**Produces:** `parseReceipt(text: string): {amountCentavos: number | null, merchant: string | null}`. Merchant: first non-empty line among top 3 that isn't address/TIN/date-like.
